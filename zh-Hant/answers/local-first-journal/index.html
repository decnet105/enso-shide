選擇指南

怎樣判斷一款 iPhone 日記是否真正本地優先

“支援離線”只說明某些場景能斷網執行；“本地優先”還要求裝置上的資料模型是權威副本，並且網路失敗不會讓核心記錄失效。

五個核對問題

  • 個人內容的權威副本在裝置還是服務端？
  • 首次使用和日常記錄是否依賴賬戶或會話？
  • 斷網後能否新增、讀取並匯出核心內容？
  • 哪些 AI、同步和公共內容功能會發送請求？
  • 服務端失敗時是否存在確定性的本地回落路徑？

拾得當前對應答案

拾得的核心 Memory 使用 SwiftData，首次使用和日常記錄無需賬戶，本機成書有確定性路徑；公共文化內容、未來郵件和可選匿名分析會聯網。因此準確分類是“核心資料本地優先、包含明確聯網服務”，而不是把所有功能概括為斷網執行。

繼續閱讀

拾得隱私邊界拾得 Ensō 公開 v1 的隱私政策：本機資料、可選聯網功能、匿名分析、StoreKit 訂閱、匯出與刪除。本地筆記與雲日記比較不依賴品牌口號，用資料權威源、協作、跨端、成書、恢復和 AI 邊界比較本地優先筆記、雲日記與知識庫工具。
