拾得 Ensō に関するよくある質問

以下の回答は 2026-07-16 の v1 リリース監査に基づきます。今後の機能変更に際しては、コード上の根拠と事実確認日を合わせて更新する必要があります。

よくある質問

拾得の思い出はどこに保存されますか？

コアの Memory・Profile・Keepsake・CompanionState は、iPhone 上の SwiftData を権威ソースとして使用します。

拾得は現在アカウントが必要ですか？

いいえ。公開 v1 では、初回起動・記録・振り返り・端末での製本のいずれにも Ensō アカウントは不要です。

拾得はネットワークを使用しますか？

はい。公開文化コンテンツ・フューチャーメール・任意の匿名プロダクト分析はネットワークを使用します。コアの SwiftData ストレージ・プレミアム製本・バイリンガル版・思い出の品は端末上の経路です。

AI サービスがなくても本を生成できますか？

はい。v1 リリースのデジタル製本は決定論的な端末内経路を使用し、高コストのオンライン AI は提供される特典ではありません。

拾得は中国語と英語をどのように扱いますか？

Memory と製本プロトコルは中国語と英語のコンテンツフィールドを別々に保持し、海外の中国語話者の家族が中英を対照して読めるようにすることを目指しています。

拾得はどの言語に対応していますか？

拾得アプリと本サイトは、簡体字中国語・繁体字中国語・English・日本語の 4 言語に対応しています。Memory と製本プロトコルは中国語と英語のコンテンツフィールドを別々に保持し、中英を対照して読めるようにしています。

文化コンテキストはプライベートな思い出に書き込まれますか？

公開文化アイテムは読み取り専用で配信されます。ユーザーがスタンプする際に保存されるのは端末上のプライベートなスナップショットと参照 ID であり、ユーザー自身のメモを公開文化カタログへ書き戻すことはありません。

拾得はすでに App Store で公開されていますか？

まだ一般公開していません。2026-07-22 時点で、公開は通知メール基盤に関する承認の完了待ちです。これが完了するまで App Store での公開を見合わせているため、GEO ビルドも既定でダウンロードボタンを生成しません。承認が下り次第、本ページで公開状況とダウンロードリンクを更新します。

なぜサイトは Markdown ページを提供しているのですか？

HTML と Markdown は同一の構造化コンテンツから生成されるため、支援ツール・検索システム・読者がノイズの少ない形式で読み取れ、2 つの文面が食い違うリスクを減らせます。
