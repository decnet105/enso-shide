# 关于拾得 Ensō 的常见问题

下面的回答以 2026-07-16 的 v1 Release 审计为准。后续能力变化需要同步更新代码证据与事实核验日期。

## 常见问题

### 拾得的回忆保存在哪里？

核心 Memory、Profile、Keepsake 和 CompanionState 以 iPhone 上的 SwiftData 为权威源。

### 拾得当前需要账户吗？

不需要。公开 v1 从首跑、记录、回看到本机成书都不要求 Ensō 账户。

### 拾得是否会使用网络？

会。公共文化内容、未来邮件和可选匿名产品分析会使用网络；核心 SwiftData 存储、高级成书、双语版本和信物是端上路径。

### 没有 AI 服务时还能生成书吗？

可以。v1 Release 的数字成书使用确定性的本机路径，高成本在线 AI 不作为已交付权益。

### 拾得如何处理中文与英文？

Memory 和成书协议保留中文与英文内容字段，目标是支持海外华语家庭的中英对照阅读。

### 拾得支持哪些语言？

拾得 App 与官网提供简体中文、繁體中文、English、日本語四种语言。Memory 与成书协议同时保留中文与英文内容字段，方便海外华语家庭做中英对照阅读。

### 文化坐标会写进私人回忆吗？

公共文化条目只读下发。用户盖印时保存的是端上私域快照和关联标识，不应把用户自述回写公共文化库。

### 拾得已经在 App Store 上线了吗？

尚未公开上线。截至 2026-07-22，App 的上线仍在等待一项通知邮件基建的审批完成；在这一步通过前，我们暂不放出 App Store 下载入口，因此 GEO 构建默认也不生成下载按钮。审批通过后，本页会更新上线状态与下载链接。

### 为什么官网提供 Markdown 页面？

HTML 与 Markdown 由同一结构化内容生成，方便辅助工具、搜索系统和读者以低噪声格式读取，并降低两套文案不一致的风险。
