# 本地优先不等于没有网络

隐私宣传必须对应实际数据路径。拾得的核心记忆模型以本机为权威源，但当前版本仍包含账户、公共内容、在线陪聊、AI 成书和旧个人事件同步路径。

## 端上保存的核心数据

- Memory：回忆正文、日期、媒体引用、心象、地点和盖印快照。
- Profile：出生年份与地点等个人资料。
- Keepsake：年度信物快照。
- CompanionState：陪伴状态与本地派生分值。

## 当前会使用网络的路径

| 路径 | 用途 | 公开时应如何说明 |
| --- | --- | --- |
| 账户接口 | 注册、登录和会话 | 当前需要账户登录 |
| 文化 API | 只读获取公共文化条目 | 公共内容联网 |
| 在线陪聊 | 服务端模型生成回复 | 在线功能 |
| AI 成书 | 服务端生成受约束的书稿结构 | 可选联网轨 |
| 旧个人事件 | 历史同步链路 | 待隐私审计与收束 |

## 当前不会公开承诺的事项

在账户、旧同步、上传和模型请求完成端到端数据流审计之前，官网不会把整个应用描述成无网络产品，也不会把架构愿景写成已经通过外部审计的结论。

> 最稳妥的公开措辞是：核心记忆以端上 SwiftData 为权威源，本机成书路径在设备上运行；部分服务需要账户和网络。
